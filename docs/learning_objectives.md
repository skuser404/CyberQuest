# Learning Objectives

## Primary Educational Goal
**Transform beginners into cybersecurity-aware individuals capable of protecting themselves and their organizations from common digital threats.**

## Bloom's Taxonomy Alignment

### Level 1: Remember (Knowledge)
**Students will be able to recall:**
- Definition of phishing, malware, social engineering
- Components of a strong password
- Meaning of HTTPS and padlock icon
- Purpose of two-factor authentication
- Basic encryption concepts

### Level 2: Understand (Comprehension)
**Students will be able to explain:**
- Why certain emails are phishing attempts
- How social engineering manipulates victims
- The difference between HTTP and HTTPS
- Why password reuse is dangerous
- How malware spreads through systems

### Level 3: Apply (Application)
**Students will be able to:**
- Identify phishing emails in real scenarios
- Create strong, unique passwords
- Verify website authenticity before entering credentials
- Recognize social engineering tactics in daily life
- Choose appropriate 2FA methods

### Level 4: Analyze (Analysis)
**Students will be able to:**
- Compare security risks across different scenarios
- Distinguish between legitimate and fake security warnings
- Evaluate password strength using multiple criteria
- Assess the trustworthiness of links and domains
- Analyze their own security vulnerabilities

### Level 5: Evaluate (Evaluation)
**Students will be able to:**
- Judge which security practices are most critical
- Critique their own security behaviors
- Prioritize security improvements based on risk
- Assess organizational security policies
- Recommend security enhancements

### Level 6: Create (Synthesis)
**Students will be able to:**
- Develop personal security protocols
- Design secure password systems using managers
- Build security awareness in their communities
- Create incident response plans for breaches
- Formulate security training for others

## Specific Learning Outcomes by Category

### 1. Phishing Detection

**Knowledge Objectives:**
- Define phishing and its variations (spear phishing, whaling, vishing)
- Identify common phishing indicators (urgency, threats, suspicious links)
- Recognize email spoofing and domain typosquatting

**Skills Objectives:**
- Hover over links to inspect URLs before clicking
- Verify sender email addresses for authenticity
- Check for spelling/grammar errors in suspicious emails
- Report phishing attempts to appropriate authorities

**Behavioral Objectives:**
- Develop skepticism toward unsolicited requests
- Pause and verify before clicking any link
- Never provide credentials via email or text
- Share phishing warnings with colleagues/family

**Assessment Criteria:**
- 80%+ accuracy identifying phishing emails
- Can explain 3+ red flags in suspicious emails
- Demonstrates verification behavior before clicking

### 2. Password Security

**Knowledge Objectives:**
- Understand password entropy and complexity
- Learn about password cracking techniques (brute force, dictionary attacks)
- Recognize the dangers of password reuse
- Know password manager benefits and usage

**Skills Objectives:**
- Create passwords with 12+ characters, mixed case, numbers, symbols
- Use password managers to generate and store unique passwords
- Implement passphrases for memorability and strength
- Set up password recovery options securely

**Behavioral Objectives:**
- Stop reusing passwords across accounts
- Change passwords after breach notifications
- Enable 2FA on all critical accounts
- Never share passwords, even with trusted individuals

**Assessment Criteria:**
- Can create passwords scoring 80+ on strength meters
- Explains why "P@ssw0rd" is weak despite complexity
- Commits to using unique passwords per account

### 3. Social Engineering

**Knowledge Objectives:**
- Define social engineering and psychological manipulation
- Identify common tactics (pretexting, baiting, quid pro quo, tailgating)
- Understand authority exploitation and urgency creation
- Recognize phone scams (vishing) and SMS scams (smishing)

**Skills Objectives:**
- Verify caller/requester identity through independent channels
- Recognize emotional manipulation attempts
- Challenge requests that seem unusual or urgent
- Protect sensitive information in conversations

**Behavioral Objectives:**
- Always verify before providing information
- Question authority when requests seem suspicious
- Slow down when feeling pressured or rushed
- Report suspicious contacts to security teams

**Assessment Criteria:**
- Identifies 3+ social engineering tactics in scenarios
- Demonstrates proper verification procedures
- Shows healthy skepticism toward unusual requests

### 4. Safe Browsing Practices

**Knowledge Objectives:**
- Understand HTTPS vs HTTP and encryption
- Learn about SSL/TLS certificates and padlock icons
- Recognize malicious websites and typosquatting
- Know safe download practices and file verification

**Skills Objectives:**
- Verify HTTPS before entering sensitive information
- Check certificate details for legitimacy
- Identify suspicious URLs and redirects
- Use browser security features and extensions
- Recognize fake security warnings (scareware)

**Behavioral Objectives:**
- Always use HTTPS for sensitive transactions
- Avoid clicking ads or pop-ups
- Download software only from official sources
- Clear browser data regularly

**Assessment Criteria:**
- Correctly identifies secure vs insecure websites 90%+
- Explains HTTPS importance in 1-2 sentences
- Demonstrates safe browsing habits

### 5. Malware Awareness

**Knowledge Objectives:**
- Define malware types (virus, worm, trojan, ransomware, spyware, adware)
- Understand infection vectors (email attachments, downloads, USB drives)
- Learn about ransomware extortion tactics
- Know signs of infection (slow performance, pop-ups, unauthorized changes)

**Skills Objectives:**
- Scan files before opening attachments
- Use reputable antivirus/anti-malware software
- Recognize fake security warnings (scareware)
- Isolate infected systems from networks
- Restore from backups after ransomware

**Behavioral Objectives:**
- Never open attachments from unknown senders
- Keep software and OS updated (security patches)
- Maintain offline backups of critical data
- Disconnect immediately if infection suspected

**Assessment Criteria:**
- Identifies 4+ malware types and behaviors
- Demonstrates proper incident response (disconnect, report, don't pay)
- Commits to regular backups and updates

### 6. Two-Factor Authentication (2FA)

**Knowledge Objectives:**
- Understand authentication factors (something you know/have/are)
- Compare 2FA methods (SMS, authenticator apps, hardware keys, biometrics)
- Learn about SIM swapping and SMS vulnerabilities
- Recognize the security benefits of 2FA

**Skills Objectives:**
- Enable 2FA on email, banking, and social media accounts
- Set up authenticator apps (Google Authenticator, Authy)
- Use backup codes securely
- Choose appropriate 2FA methods for different accounts

**Behavioral Objectives:**
- Enable 2FA on all accounts that support it
- Prioritize hardware keys or app-based 2FA over SMS
- Store backup codes securely offline
- Never share 2FA codes with anyone

**Assessment Criteria:**
- Can rank 2FA methods by security level
- Explains why SMS 2FA is vulnerable to SIM swapping
- Has enabled 2FA on at least 3 accounts

### 7. Encryption Basics (Advanced)

**Knowledge Objectives:**
- Understand symmetric vs asymmetric encryption
- Learn about end-to-end encryption (E2EE)
- Know about VPNs and encrypted connections
- Recognize the importance of encryption for privacy

**Skills Objectives:**
- Use encrypted messaging apps (Signal, WhatsApp)
- Enable full disk encryption on devices
- Use VPNs on public WiFi networks
- Verify encryption indicators in apps

**Behavioral Objectives:**
- Prefer encrypted communication for sensitive topics
- Use VPNs when on untrusted networks
- Enable device encryption on phones and laptops
- Choose services that offer E2EE

**Assessment Criteria:**
- Explains E2EE in simple terms
- Can name 2+ encrypted messaging apps
- Understands when VPNs are necessary

### 8. Data Privacy (Advanced)

**Knowledge Objectives:**
- Understand data collection by apps and websites
- Learn about privacy policies and user agreements
- Know about GDPR, CCPA, and privacy rights
- Recognize over-permissioning by mobile apps

**Skills Objectives:**
- Review and limit app permissions
- Use privacy-focused browsers and search engines
- Configure privacy settings on social media
- Opt out of data sharing when possible

**Behavioral Objectives:**
- Read permission requests before accepting
- Regularly audit app permissions
- Minimize personal information shared online
- Delete unused accounts and apps

**Assessment Criteria:**
- Identifies unnecessary permissions in apps
- Can explain why privacy matters
- Takes action to limit data exposure

### 9. Incident Response (Advanced)

**Knowledge Objectives:**
- Understand incident response steps (identify, contain, eradicate, recover)
- Learn about reporting procedures (IT, authorities, credit bureaus)
- Know about breach notification laws
- Understand the importance of honesty in reporting

**Skills Objectives:**
- Disconnect infected devices from networks
- Document incident details for reporting
- Change passwords from clean devices
- Monitor accounts for unauthorized activity
- File reports with appropriate authorities

**Behavioral Objectives:**
- Report incidents immediately without shame
- Follow organizational incident response procedures
- Cooperate fully with security investigations
- Learn from incidents to prevent recurrence

**Assessment Criteria:**
- Lists 4+ incident response steps in order
- Demonstrates proper reporting procedures
- Shows no hesitation to report mistakes

## Cross-Cutting Skills

### Critical Thinking
- Question unusual requests or communications
- Verify information through multiple sources
- Analyze risks before taking actions

### Digital Literacy
- Navigate security settings across platforms
- Understand basic technical concepts
- Use security tools effectively

### Risk Assessment
- Evaluate the sensitivity of information
- Prioritize security measures based on threats
- Balance convenience with security

### Communication
- Explain security concepts to non-technical people
- Report incidents clearly and accurately
- Share security knowledge with others

## Progression Model

### Beginner Level (Foundation)
**Goal**: Build fundamental awareness and recognize obvious threats

**Focus Areas**:
- Basic phishing recognition
- Password strength fundamentals
- HTTPS verification
- Common social engineering tactics

**Expected Outcome**: Can protect self from most common attacks

### Intermediate Level (Application)
**Goal**: Apply knowledge to realistic scenarios and develop good habits

**Focus Areas**:
- Advanced phishing (spear phishing with correct branding)
- Password management strategies
- Public WiFi risks and VPN usage
- Credential stuffing and breach response

**Expected Outcome**: Makes informed security decisions daily

### Advanced Level (Mastery)
**Goal**: Understand sophisticated threats and teach others

**Focus Areas**:
- Zero-day vulnerabilities
- End-to-end encryption
- Data privacy and over-permissioning
- Ransomware response and backups
- Incident response procedures

**Expected Outcome**: Becomes security advocate in community

## Assessment & Feedback Strategy

- Category-based performance tracking
- Progress visualization

### Summative Assessment (End of Game)
- Overall risk level assessment
- Category breakdown (strengths/weaknesses)
- Personalized improvement suggestions
- Comparison with other players (leaderboard)

### Metacognitive Reflection
- "What surprised you most?"
- "Which threat do you now take more seriously?"
- "What will you change in your daily behavior?"

## Real-World Application

### Immediate Actions (After Playing)
1. Enable 2FA on email and banking accounts
2. Change reused passwords to unique ones
3. Install a password manager
4. Verify HTTPS before entering credentials
5. Report suspicious emails as phishing

### Medium-Term Actions (Within 1 Month)
1. Audit all account security settings
2. Set up encrypted backups
3. Train family members on basics
4. Use VPN for public WiFi
5. Review app permissions

### Long-Term Behaviors (Ongoing)
1. Stay informed about new threats
2. Share security knowledge with others
3. Participate in security training at work
4. Advocate for better security practices
5. Help elderly/vulnerable users stay safe
